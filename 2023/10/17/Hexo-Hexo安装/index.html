前置条件

  1. Node.js
  2. Git
  3. Npm
  4. GitHub账号
  5. 个人域名(可选)

前置条件安装配置

Node.js安装

Node.js官网下载：Node.js
Node.js安装教程：Node.js安装与配置（详细步骤）_nodejs安装及环境配置_liyitongxue的博客-CSDN博客
版本要求：Node.js 版本需不低于 10.13，建议使用 Node.js 12.0 及以上版本

安装时勾选该选项
“自动安装必要的工具，注意：这也将安装Chocolatey。安装完成后，该脚本将弹出在一个新窗口中。”
会自动添加PATH（环境变量)，但是弹出的窗口会安装大量内容，不推荐

推荐：不勾选该选项，则需要手动配置环境变量

配置环境变量

win+i键打开设置，点击“系统”，找到“高级系统设置”点击“环境变量”，编辑【系统变量】中的变量【Path】
粘贴Node.js的安装路径

安装验证

此版本node.js集成了npm，可以在命令行中检查是否安装成功。

Git安装

Git官网下载：Git - Downloads
Git安装教程：[Git & GitHub] Windows安装git和环境变量配置_配置git环境变量-CSDN博客
选择Windows系统的Git

点击下载64位最新版本

双击安装程序，点击Next，再选择安装路径，安装路径自定义。

这些选项都可以勾选
选项Additional icons：额外图标（添加桌面快捷方式）
第二项是资源管理器中的右键菜单
Bash是Git的命令行，而GUI则是图形化界面

创建开始菜单

下一步，这里我选择“使用VS Code作为Git的默认编辑器”，其他的如Notepad也是可以的，看个人喜好而定

使用Git自带的命令行工具

使用OpenSSL传输加密协议

选择换行格式，点击Next

  • 让Git能够自动转换文件中的换行符：签出到本地时转换为Windows下的换行符，提交到服务器时转换为Unix下的换行符
  • 让Git在签出到本地时不做转换，保留原始文件的换行符；提交到服务器时转换为Unix下的换行符
  • 让Git在签出到本地时和提交到服务器时都不做转换


配置Git bash终端仿真器，点击Next

  • 使用MinTTY终端
  • 使用windows默认的命令行







完成安装
环境变量可配置可不配置，并不会非常频繁地在cmd中使用到Git，一般都是通过在文件夹内右键选择“Git Bash Here”来打开git的命令行。

验证安装

GitHub配置

若无GitHub账号，则注册一个
GitHub账号注册点击注册

输入你的邮箱
之后再输入密码

再输入可用的用户名

选择yes 或 no来选择是否接收GitHub的推送，推荐选择yes

之后再验证真人，就能成功创建账号了。
新建一个存储库

存储库名为：
你的用户名.github.io

存储库创建成功

配置SSH免密登录

配置缘由部署网站代码需要GitHub权限，直接使用用户名和密码不安全，且繁琐。
所以我们使用 ssh key 来解决本地和服务器的连接问题。

注: 若配置过 SSH，可跳过此步骤
步骤:

  1. 删除C:\Users\Administrator\.ssh路径下的文件（可选），可以不删。
  2. WIN+R键输入cmd打开命令行，输入命令ssh-keygen -t rsa -C "你的GitHub邮箱"生成SSH公钥，之后连敲三次回车，出现下图情况代表成功

  1. 生成了一个新的 C:\Users\您的用户名\ .ssh文件夹，打开这个文件夹，找到id_rsa.pub文件，记事本打开并复制里面的内容

  1. 打开GitHub主页，进入设置 -> SSH and GPG keys -> New SSH key，将复制的公钥粘贴到下方出现的输入框内，title 随便填，保存即可。设置好如下图:

  1. 检测是否设置成功:

输入命令: ssh -T git@github.com

看到以上信息说明 SSH 已配置成功!

绑定你自己的域名（可选）

如果你已经拥有你自己的域名的话那是最好，如果你想购买一个域名，可以去腾讯云，阿里云，或者godaddy购买。
由于我的域名是在godaddy购买的，接下来我会演示如何绑定你的github pages到godaddy，其他域名商操作应该也是雷同的。
登录你的godaddy后台，点击manage DNS

添加两条记录，一条CNAME，一条A记录，如图：

其中CNAME的value就填写你的github page的值，A的value可以通过ping http://xxx.github.io得到。
接下来，去到你的github page的settings界面，在Options->Github pages->custom domain添加你的域名：

最后一步，在你本地 /source 文件夹下添加一个名为CNAME的文件（注意，此文件没有后缀），里面添加你的域名即可。

再次执行deploy
